# O Projeto GNU

O **Projeto GNU** foi lançado em 1983 por Richard Stallman com o objetivo de criar um sistema operacional completo e totalmente livre, que respeitasse a liberdade dos usuários. O nome "GNU" é um acrônimo recursivo que significa "GNU's Not Unix!" (GNU Não é Unix!), uma forma de reconhecer que o sistema foi projetado para ser semelhante ao Unix, mas sem usar seu código.

### A Filosofia do Software Livre

O pilar central do projeto GNU é a filosofia do **software livre**. Um software é considerado "livre" se garante aos usuários quatro liberdades essenciais:

1.  A liberdade de **executar** o programa para qualquer propósito.
2.  A liberdade de **estudar** como o programa funciona e adaptá-lo às suas necessidades (acesso ao código-fonte é um pré-requisito).
3.  A liberdade de **redistribuir** cópias para ajudar outros.
4.  A liberdade de **distribuir** cópias de suas versões modificadas para os outros, dando a toda a comunidade a chance de se beneficiar de suas mudanças.

### Componentes do Sistema GNU

Para criar um sistema operacional funcional, o projeto GNU desenvolveu uma vasta coleção de componentes de software, incluindo:

*   **GCC (GNU Compiler Collection):** Um conjunto de compiladores essencial para o desenvolvimento de software.
*   **Bash (Bourne-Again Shell):** O shell de linha de comando padrão na maioria das distribuições Linux.
*   **Coreutils:** Um pacote com as ferramentas básicas de linha de comando (`ls`, `cp`, `mv`, `rm`, etc.).
*   **GNOME:** Um dos ambientes de desktop mais populares.

### A Parceria com o Linux

No início da década de 1990, o sistema GNU estava quase completo, mas ainda faltava uma peça crucial: o **kernel**, o componente de baixo nível que interage diretamente com o hardware.

Foi nesse ponto que o kernel **Linux**, criado por Linus Torvalds, surgiu. Como o Linux foi lançado sob a Licença Pública Geral GNU (GPL), ele pôde ser combinado com o sistema GNU para formar um sistema operacional completo e funcional: o **GNU/Linux**.
