# Etiqueta em Pull Requests

Pull Requests (PRs) são o coração da colaboração em código. Seguir uma etiqueta adequada agiliza a revisão e mantém o ambiente saudável.

## Para Contribuidores

- **Seja Descritivo**: Explique claramente *o que* mudou e *por que*. Use templates se disponíveis.
- **Commits Atômicos**: Mantenha commits focados em uma única tarefa lógica.
- **Paciência**: Mantenedores são voluntários. Aguarde o feedback com paciência e educação.
- **Responda aos Comentários**: Agradeça o feedback e discuta as sugestões abertamente.

## Para Revisores

- **Seja Gentil**: Aponte erros com cortesia. Sugira melhorias em vez de apenas criticar.
- **Reconheça o Esforço**: Agradeça pela contribuição, mesmo se o PR não for aceito.
- **Seja Claro**: Se algo precisa ser mudado, explique o motivo técnico claramente.

Um bom fluxo de PR é uma conversa técnica onde todos aprendem e o projeto ganha qualidade.
