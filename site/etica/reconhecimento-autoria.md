# Reconhecimento e Autoria

Dar crédito a quem merece é a base da etiqueta no mundo open source e acadêmico. O reconhecimento motiva e legitima o trabalho alheio.

## Como Dar Crédito

- **Co-autoria em Commits**: Use tags como `Co-authored-by:` no rodapé das mensagens de commit para creditar pair programming ou sugestões.
- **Agradecimentos (Hall of Fame)**: Mantenha uma lista de contribuidores no `README` ou em um arquivo `CONTRIBUTORS`.
- **Derivações**: Se seu projeto é um fork ou usa código de outro, mencione a origem explicitamente.

## Plágio vs Inspiração

- **Cópia sem Atribuição**: Copiar código ou texto e apresentar como seu é plágio e desonesto.
- **Inspiração**: Ler o código de alguém para entender a lógica e implementar a sua própria versão é válido, mas citar a fonte da ideia é uma boa prática ("Inspirado pela implementação de X").

## Licenças

Respeitar a licença é a forma legal de reconhecimento. Mantenha os cabeçalhos de copyright originais quando exigido pela licença (ex: MIT, BSD, GPL).
