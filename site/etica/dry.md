# Don't Repeat Yourself, DRY, `dry`

O conceito do DRY, ou "Don't Repeat Yourself" (Não se Repita), é um princípio fundamental no desenvolvimento de software que visa reduzir a repetição de código. A ideia central é que cada parte do conhecimento em um sistema deve ter uma representação única, não ambígua e autoritativa. Na prática, isso significa evitar a duplicação de código, lógica de negócios ou qualquer outra informação.

Quando o código é duplicado, a manutenção se torna mais difícil e propensa a erros. Se uma alteração for necessária, ela precisará ser feita em todos os lugares onde o código foi repetido. Se um local for esquecido, o sistema se tornará inconsistente. Ao seguir o princípio DRY, você pode criar sistemas mais fáceis de manter, entender e modificar.
