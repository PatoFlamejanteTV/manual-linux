# Boas Práticas em Issues

Issues são ferramentas de trabalho, mas também canais de comunicação. Usá-las bem demonstra respeito pelo tempo de todos.

## Antes de Abrir

- **Pesquise**: Verifique se o problema já foi reportado. Duplicatas geram ruído.
- **Reproduza**: Tenha certeza de que o bug é reproduzível e isole o cenário.
- **Leia o Guia de Contribuição**: Cada projeto tem suas regras (ex: uso de templates).

## Ao Escrever

- **Título Claro**: Resuma o problema em poucas palavras.
- **Passo a Passo**: Liste como reproduzir o erro:
  1. Faço isso...
  2. Acontece aquilo...
  3. Esperava aquilo outro...
- **Ambiente**: Informe SO, versão do software, navegador, etc.
- **Seja Polido**: Lembre-se que do outro lado há uma pessoa. "Isso não funciona" é vago; "Encontrei um comportamento inesperado ao fazer X" é construtivo.

## Acompanhamento

- Se você resolver o problema sozinho, feche a issue e explique a solução.
- Se o mantenedor pedir mais info, responda prontamente.
